# Промпт для Lovable: Сайт рецептов CookBook

## Общее описание проекта

Создай современный, красивый и функциональный сайт рецептов с полным набором функций для публикации, просмотра и взаимодействия с кулинарными рецептами. Дизайн должен быть современным, чистым, с акцентом на визуальную привлекательность фотографий блюд.

## Технический стек

- Frontend: React + TypeScript + Tailwind CSS
- Backend API: FastAPI (уже существует, нужно интегрироваться)
- Аутентификация: JWT токены
- Хранение файлов: публичные URL для изображений

## Основные страницы и функциональность

### 1. Авторизация и регистрация

**Страница входа (Login)**
- Форма входа с email и паролем
- Кнопка "Забыли пароль?"
- Ссылка на регистрацию
- Валидация полей
- Обработка ошибок с понятными сообщениями

**Страница регистрации (Register)**
- Форма: email, пароль, подтверждение пароля, nickname (опционально)
- Валидация: email формат, минимальная длина пароля, совпадение паролей
- После регистрации - автоматический вход
- Обработка ошибок (email уже используется и т.д.)

**Восстановление пароля**
- Страница "Забыли пароль?" - ввод email
- Страница сброса пароля по токену

### 2. Главная страница (Home)

**Основные элементы:**
- Заголовок "Рецепты" или "CookBook"
- Поисковая строка вверху (полнотекстовый поиск по названию и описанию)
- Фильтры и сортировка:
  - Фильтр по времени приема пищи: "Все", "Завтрак", "Обед", "Ужин" (табы или кнопки)
  - Сортировка: "Новые первыми", "Старые первыми", "По популярности" (по количеству лайков)
- Сетка рецептов (адаптивная: 1 колонка на мобильных, 2 на планшетах, 3 на десктопе)

**Карточка рецепта:**
- Обложка (фото рецепта) - занимает верхнюю часть карточки, высота ~200px, object-cover
- Название рецепта (кликабельное, ведет на детальную страницу)
- Автор рецепта (аватарка + nickname/email, кликабельно)
- Категория (завтрак/обед/ужин) и дата публикации
- Кнопка лайка (сердечко) с счетчиком лайков
- При клике на карточку - открывается детальная страница в новой вкладке

**Дополнительно:**
- Бесконечная прокрутка или пагинация
- Состояние загрузки (скелетоны или спиннер)
- Пустое состояние, если рецептов нет

### 3. Детальная страница рецепта (Recipe Detail)

**Открывается в новой вкладке при клике на рецепт**

**Структура:**
- Большое фото рецепта (обложка) - на всю ширину, высота ~400-500px
- Название рецепта (крупный заголовок)
- Метаинформация: категория, дата, автор (с аватаркой и ссылкой на профиль)
- Кнопка лайка с счетчиком (если авторизован)
- Кнопка "Редактировать" (только для автора рецепта)
- Описание рецепта (если есть)

**Ингредиенты:**
- Отдельная секция с фоном или рамкой
- Список ингредиентов: название - количество
- Красивое оформление списка

**Пошаговое приготовление:**
- Каждый шаг в отдельной карточке/блоке
- Номер шага (Шаг 1, Шаг 2...)
- Текст инструкции
- Фото шага (если есть) - под текстом, адаптивное
- Шаги идут последовательно, визуально разделены

**Комментарии:**
- Секция комментариев внизу страницы
- Список комментариев:
  - Аватарка автора
  - Имя автора (nickname или email)
  - Текст комментария
  - Дата комментария
  - Кнопки "Изменить" и "Удалить" (для автора комментария или автора рецепта)
- Форма добавления комментария (только для авторизованных):
  - Текстовое поле
  - Кнопка "Отправить"
- Редактирование комментария inline (при клике на "Изменить")

**Дополнительно:**
- Кнопка "Назад" или хлебные крошки
- Состояние загрузки
- Обработка ошибок (рецепт не найден)

### 4. Личный кабинет / Профиль (Profile)

**Мой профиль (для авторизованного пользователя):**

  - Кнопка "Создать новый рецепт"
  - Для каждого рецепта: кнопки "Редактировать" и "Удалить"

**Публичный профиль пользователя:**

**Форма создания/редактирования:**
- Название рецепта (обязательное поле)
- Описание (многострочное текстовое поле, опционально)
- Категория: выбор из "Завтрак", "Обед", "Ужин" (радио-кнопки или селект)
- Загрузка обложки (фото рецепта):
  - Превью загруженного фото
  - Кнопка загрузки/замены
  - Поддержка drag & drop

  - Динамический список: можно добавлять/удалять ингредиенты

  - Динамический список шагов
  - Для каждого шага:
    - Текстовое поле для инструкции
    - Загрузка фото для шага (опционально)

    - Кнопка удаления шага

  - Возможность переупорядочивания шагов (drag & drop или стрелки вверх/вниз)

- Валидация: название обязательно, минимум один ингредиент, минимум один шаг

## Дизайн и UX требования

### Цветовая схема:
- Основной цвет: теплый, аппетитный (например, оранжевый/красный для акцентов)
- Фон: светлый, чистый (белый или очень светло-серый)
- Текст: темно-серый/черный для читаемости
- Акценты: использовать основной цвет для кнопок, ссылок, активных состояний

### Типографика:
- Четкая иерархия заголовков
- Читабельные шрифты
- Достаточные отступы между элементами

### Компоненты:
- Карточки рецептов: скругленные углы, тень для глубины
- Кнопки: четкие, с hover-эффектами
- Формы: понятные labels, валидация в реальном времени
- Загрузка: скелетоны или спиннеры
- Модальные окна: для подтверждения удаления

### Адаптивность:
- Мобильные устройства: 1 колонка, упрощенная навигация
- Планшеты: 2 колонки
- Десктоп: 3+ колонки, полная функциональность

### Анимации:
- Плавные переходы при hover
- Анимация загрузки контента
- Плавное появление карточек при скролле

## API интеграция

Backend API уже существует на FastAPI. Основные endpoints:

- `POST /auth/register` - регистрация
- `POST /auth/login` - вход
- `GET /recipes` - список рецептов (query params: topic, order, q, limit, offset)
- `GET /recipes/popular` - популярные рецепты
- `GET /recipes/{id}` - детали рецепта
- `POST /recipes` - создание рецепта (multipart/form-data)
- `PUT /recipes/{id}` - редактирование рецепта
- `DELETE /recipes/{id}` - удаление рецепта
- `POST /recipes/{id}/like` - лайк/анлайк
- `GET /recipes/{id}/comments` - комментарии к рецепту
- `POST /recipes/{id}/comments` - добавить комментарий
- `PUT /recipes/{id}/comments/{comment_id}` - редактировать комментарий
- `DELETE /recipes/{id}/comments/{comment_id}` - удалить комментарий
- `GET /users/me` - текущий пользователь
- `GET /users/{id}` - публичный профиль

Все запросы к защищенным endpoints должны включать JWT токен в заголовке `Authorization: Bearer {token}`.
